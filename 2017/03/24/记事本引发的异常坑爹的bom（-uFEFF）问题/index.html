记事本引发的异常坑爹的bom（﻿）问题

  • php
  • windows

两天前。学校一台服务器坏了。没错，就是坏了，单点，没错，没有任何备份！可怕。修完机器后，我的两个小项目让我重新部署，维护的老师直接丢了账号密码给我，自己搞。环境什么的。算了呗，反正也搞过。就是没想到遇到一个问题搞了2 3个钟。。。已吐血。
其实标题（）中是有东西的，就是这红点的符号\uFEFF

服务器环境： window server 2008。。。

简单暴力，直接下载phpstudy搞定。。。

坑来了

起因：由于服务器上没有任何编辑器，我想直接丢上代码就算了呗，赶紧搞定，直接在服务器上改点配置就完了，没错，用的记事本。记事本！

前端ajax请求后台接口正常返回数据

本地一切正常，但由于返回的json格式数据自动加上了红点，jq一直报错json格式不正确。后来在控制台response看到了 { 前面莫名其妙多了一个红点，一开始没注意，真是ri了dog。。（后面解决后效果重现不出来。红点如下图，别在意其他代码。）

解决后，想重现下bom效果，重新上传整个项目代码上服务器，在服务器上用记事本打开AdminController.php这个文件后关闭，不改动任何代码。
结果报错如下：

可以啊这报错猝不及防。正常会以为真的是namespace前面多写了其他东西才报的错，但事实没有，而且bom符号代码文件中并不可见，然后沿着这问题找下去，开始傻比一样一直找。找到天荒地老。。仍旧一脸dog逼

最终得出结论，千万别用记事本编辑PHP。。。。。
多么痛的领悟。。。
bom和这个符号\uFEFF是什么请自行百度谷歌
